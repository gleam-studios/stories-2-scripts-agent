# 快捷指令规范

一、目的

本文件规定这个 `agent` 工作区在对话里的快捷指令约定。

这里的快捷指令不是客户端原生命令，而是本工作区内部固定采用的对话触发规则。

二、固定支持的快捷指令

当前只固定支持：

1. `/start`
2. 自动收尾

三、`/start` 规则

当用户输入：

```text
/start
```

总控 agent 必须立即返回：

```text
请提供以下信息，我将按 agent 工作区标准流程启动单篇项目：

项目代号：<project_slug>
剧名：<project_title>
原文文件：<source_file_path>
目标市场：<target_market，默认 NA-English>
目标集数：<episode_count，可暂留空>
单集时长：<episode_duration，默认 1-2 min>
其他限制：<production_constraints，可留空>
```

四、`/start` 强制要求

1. 不得直接开始写作。
2. 不得自行脑补原文路径。
3. 不得跳过 `project_slug`。
4. 不得跳过 `project_title`。
5. 必须先完成信息收集，再进入 `work/<project_slug>/` 初始化。

五、自动收尾规则

当以下条件同时满足时，必须自动收尾：

1. 英文大纲已完成
2. 英文剧本已完成
3. 英文分镜表已完成
4. 英文设定集已完成
5. 中文 4 件套已完成
6. 8 个 Word 文件已导出

六、自动收尾动作

1. 把 8 个正式 Word 文件导出到：
   - `~/Desktop/短剧导出/<project_slug>/`
2. 用固定话术回复：

```text
工作已结束，Word 已导出到桌面：
~/Desktop/短剧导出/<project_slug>/
```

七、禁止项

1. 禁止在 `/start` 后直接开跑。
2. 禁止任务完成后还要求用户再发一次“导出”命令。
3. 禁止把 `.json` 或 `.md` 工作稿当成最终正式交付。
4. 禁止缺少 `project_title` 时直接导出 Word。
