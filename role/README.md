# role 使用说明

这个目录定义三件事：

1. 这个 agent 是谁
2. 这个 agent 怎么开始
3. 这个 agent 怎么结束

## 阅读顺序

1. `快捷指令规范.md`
2. `总控Agent角色.md`
3. `单篇执行流程.md`
4. `启动与导出规范.md`

## 文件分工

`快捷指令规范.md`

- 定义 `/start`
- 定义自动收尾

`总控Agent角色.md`

- 定义 agent 的身份、边界、任务和最终交付

`单篇执行流程.md`

- 定义 8 个内容 stage 的顺序、输入输出和完成标准（含 `source-narrative-digest` 与改编锁定）

`启动与导出规范.md`

- 定义标准启动模板、导出模板、文件命名和桌面导出路径

## 一句话理解

`role/` 不是写内容的地方，而是规定这个 agent 该如何工作、如何启动、如何收尾的地方。
