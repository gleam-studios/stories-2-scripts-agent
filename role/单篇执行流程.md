单篇英文网文执行流程

一、适用范围

本流程只适用于：

单篇英文网文
-> 单次完整改编
-> 单次双语交付

当前不处理：

1. 批量队列
2. 多作品并发

多角色架构说明：

面向 API / 前端时，每个 stage 由独立子 Agent 执行，执行完后由规格审核员审核，审核通过并经用户确认后才进入下一 stage。详见 `prompts/` 目录下的 prompt 文件。

| Stage | 执行者 | 审核后动作 |
|-------|--------|-----------|
| 1. source-analysis | 原文分析师 | 规格审核员审核 → 用户确认 |
| 1b. source-narrative-digest | 原作梳理师 | 规格审核员审核 → 用户确认 |
| 2. overseas-adaptation-planner | 改编策划师 | 规格审核员审核 → 用户确认 |
| 3. english-setting-bible-builder | 设定集构建师 | 规格审核员审核 → 用户确认 |
| 4. english-outline-writer | 大纲创作师 | 规格审核员审核 → 用户确认 |
| 5. english-screenplay-writer | 剧本创作师 | 规格审核员审核 → 用户确认 |
| 6. english-storyboard-table-builder | 分镜构建师 | 规格审核员审核 → 用户确认 |
| 7. chinese-mirror-pack-translator | 镜像翻译师 | 规格审核员审核 → 用户确认 |

总制作人（`prompts/主Agent/总制作人.txt`）负责调度全流程，纯调度不生成内容。

长文本（约 ≥50 万字符或单次无法装下「原文+规则+指令」）必须遵守 `rules/16_长文本分段执行规范.md`：子步骤不断档、不跳 merge/align/stitch。

二、标准顺序

1. 输入归一化
2. `source-analysis`
3. `source-narrative-digest`（原作五维梳理 → `narrative_digest.json`）
4. `overseas-adaptation-planner`（须含改编锁定五字段 + 改名映射 + `similarity_score.total` ≤ 30，见 `rules/17`）
5. `english-setting-bible-builder`
6. `english-outline-writer`
7. `english-screenplay-writer`
8. `english-storyboard-table-builder`
9. `chinese-mirror-pack-translator`
10. 最终导出 Word

三、各阶段输入输出

1. 输入归一化

输入：

- 英文原文（路径或粘贴；长文以 `work/<project_slug>/source_raw.txt` 或等价单一文件为真源）
- 章节（**推荐** `chapter_index.yaml`：每章 `chapter_id`、标题、`start_line`/`end_line` 或 `char_start`/`char_end` 或起止锚点子串）
- 简介
- 标签
- 来源

输出：

- `source_packet.json`（元数据与指针；**不必**把百万字原文重复嵌入 JSON，可只写 `source_path` 指向 `source_raw.txt`）

**长文额外产物（推荐）：**

| 文件 | 作用 |
|------|------|
| `chapter_index.yaml` | 切包与局部引用原文 |
| `source_chunks/` | 可选，预切 `ch_XX-YY.txt` |
| `run_manifest.json` | `completed_steps`、`chunk_ids_done`、`outline_batches`、`last_episode_screenplay`、`last_storyboard_scene`、`notes` |

**续跑：** 新对话先读 `run_manifest.json` 与 `soul.md`。

1. `source-analysis`

输入：

- `source_packet`（及长文下的 `chapter_index` / `source_chunks` 与按包原文）

输出：

- `story_bible.json`（长文须经 **chunk partial → merge**，并写 `source_coverage`；见 `skills/source-analysis/SKILL.md`）

1. `source-narrative-digest`

输入：

- `story_bible`（长文禁止为补全而整书重读，见 `rules/16`）

输出：

- `narrative_digest.json`（五维原文梳理：`plot_direction` / `story_structure` / `character_backgrounds` / `character_profiles` / `relationships_and_turns`；见 `skills/source-narrative-digest/SKILL.md`）

1. `overseas-adaptation-planner`

输入：

- `story_bible`
- `narrative_digest`
输出：
- `adaptation_plan`（须含 `adapted_*` 五字段、`character_rename_map`、`all_principal_names_changed`、`similarity_score` 且 `total` ≤ 30；下游服从 `rules/17_改编计划锁定与下游服从.md`）

1. `english-setting-bible-builder`

输入：

- `story_bible`
- `adaptation_plan`
输出：
- `english_setting_bible`

1. `english-outline-writer`

输入：

- `adaptation_plan`
- `english_setting_bible`
输出：
- `english_outline_pack`

1. `english-screenplay-writer`

输入：

- `adaptation_plan`
- `english_setting_bible`
- `english_outline_pack`
输出：
- `english_screenplay_pack`

（正文须落实 `adaptation_plan` 锁定契约，见 `rules/17`。）

1. `english-storyboard-table-builder`

输入：

- `adaptation_plan`
- `english_setting_bible`
- `english_outline_pack`
- `english_screenplay_pack`
输出：
- `english_storyboard_pack`

1. `chinese-mirror-pack-translator`

输入：

- 英文设定集
- 英文大纲
- 英文剧本
- 英文分镜表
输出：
- `chinese_mirror_pack`

四、每一步最低完成标准

1. `source-analysis`

必须识别：

- 主角与反派
- 世界背景种子
- 时间线种子
- 地点候选
- 道具候选
- 视觉主题种子
- 高相似度风险

1. `source-narrative-digest`

必须给出：

- 剧情走向、故事结构、人物背景、人物设定、人物关系与转变（五维），且与 `story_bible` 的 `character_id` 对齐

1. `overseas-adaptation-planner`

必须给出：

- 开场模板
- 钩子分布
- 卡点规划
- 节奏曲线
- 爽点矩阵
- 反派梯度
- 角色功能分配
- 相似度自评分（`total` ≤ 30）
- **改编锁定**：`adapted_plot_direction`、`adapted_structure`、`adapted_background`、`adapted_character_system`、`adapted_relationships_and_turns`
- **改名映射**：`character_rename_map`，且 `all_principal_names_changed` 为真

1. `english-setting-bible-builder`

必须给出：

- 世界背景
- 时间线
- 角色详设与形象设计
- 场景设定
- 道具设定
- 服化与视觉系统
- 连续性规则
- AI 资产矩阵

1. `english-outline-writer`

必须给出：

- 改编定位
- 全季结构
- 分集大纲
- 每集钩子
- 卡点图

1. `english-screenplay-writer`

必须给出：

- 英文剧本正文
- scene crosswalk
- 不带镜头语言的 screenplay

1. `english-storyboard-table-builder`

必须给出：

- 场次索引
- shot-level rows
- 镜头、机位/运动、道具、音效、转场、AI 备注

1. `chinese-mirror-pack-translator`

必须给出：

- 中文大纲
- 中文剧本
- 中文分镜表
- 中文设定集
- 对齐说明

五、最终导出规则

最终对外只导出 8 个 Word 文件：

1. `01_<剧名>_英文大纲.docx`
2. `02_<剧名>_英文剧本.docx`
3. `03_<剧名>_英文分镜表.docx`
4. `04_<剧名>_英文设定集.docx`
5. `05_<剧名>_中文大纲.docx`
6. `06_<剧名>_中文剧本.docx`
7. `07_<剧名>_中文分镜表.docx`
8. `08_<剧名>_中文设定集.docx`

六、完成前最后核对

1. 英文 1-4 是否成立
2. 中文 5-8 是否只从英文生成
3. 是否按英文 1-4、中文 5-8 编号
4. 是否全部带 `Quick Reference`
5. 是否与样例结构和专业度一致
6. 长文：`story_bible.source_coverage` 与 `chapter_index` 一致；大纲集数对齐 `adaptation_plan.episode_count`；分镜非单次全剧生成（除非 `rules/16` 允许的极少集数）
7. `narrative_digest` 已存在且 `adaptation_plan` 含完整锁定字段；生产链未违反 `rules/17`

七、长文子步骤速查（不改变对外 stage 名称；`1b` 仍属前置梳理）

| Stage | 内部子步骤 |
|-------|------------|
| source-analysis | chunk pass → merge pass |
| source-narrative-digest | 单 pass 归纳（仅依赖 story_bible + 必要时局部 chunk） |
| english-outline-writer | batch（§1–8 + 分批评分集）→ align pass |
| english-screenplay-writer | 按集（每调用 ≤3 集）→ stitch pass |
| english-storyboard-table-builder | 按集或按场 → 密度 second pass |
| english-setting-bible-builder | 可选：look/loc/prop/ai_matrix 分批 → merge JSON |
| chinese-mirror-pack-translator | 与英文相同批次边界 → 合并 `mirror_pack` |

细则与 Continuity Sheet 定义见 `rules/16_长文本分段执行规范.md`。
