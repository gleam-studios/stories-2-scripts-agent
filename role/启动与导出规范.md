启动与导出规范

一、目的

本文件规定这个 `agent` 工作区的标准启动模板和标准收尾导出规则。

快捷指令触发方式统一参照：

`快捷指令规范.md`

二、标准启动模板

```text
请在 agent 工作区启动一个单篇项目。
项目代号：<project_slug>
剧名：<project_title>
原文文件：<source_file_path>
目标市场：<target_market>
目标集数：<episode_count>
单集时长：<episode_duration>
其他限制：<production_constraints>

要求：
1. 先在 work/<project_slug>/ 建立工作目录
2. 先生成 source_packet.json
3. 再按固定 stage 顺序推进
4. 最终只交付 8 个中文命名 Word 文件
```

补充说明：

1. `project_slug` 只用于工作目录与桌面导出目录。
2. `project_title` 用于最终 8 个 Word 文件名中的剧名字段。

三、最小启动模板

```text
请在 agent 工作区启动单篇项目：
project_slug=<project_slug>
project_title=<project_title>
source=<source_file_path>
```

如果缺失参数，默认规则如下：

1. `target_market = NA-English`
2. `episode_duration = 1-2 min`
3. `episode_count` 由 agent 先给出建议
4. `project_title` 未提供时，最终导出前必须补齐

四、启动后固定动作

1. 读取 `SKILL.md`
2. 读取 `role/` 内 4 份规范
3. 创建 `work/<project_slug>/`
4. 生成 `source_packet.json`
5. 从 `source-analysis` 开始

五、正式收尾规则

收尾前必须检查：

1. 英文 1-4 是否完整
2. 中文 5-8 是否与英文完全镜像
3. 8 个文件是否按编号顺序完整存在
4. 大纲、剧本、分镜表、设定集是否边界清楚

六、最终正式交付物

1. `01_<剧名>_英文大纲.docx`
2. `02_<剧名>_英文剧本.docx`
3. `03_<剧名>_英文分镜表.docx`
4. `04_<剧名>_英文设定集.docx`
5. `05_<剧名>_中文大纲.docx`
6. `06_<剧名>_中文剧本.docx`
7. `07_<剧名>_中文分镜表.docx`
8. `08_<剧名>_中文设定集.docx`

命名结构固定为：

`编号_剧名_文件类型.docx`

并额外统一导出到桌面目录：

`~/Desktop/短剧导出/<project_slug>/`
