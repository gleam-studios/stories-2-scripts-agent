# 12 剧本规范

## 一、定位

剧本是大纲和分镜表之间的**中间层**。

- 往前：承接大纲里的集级结构（`episode_outlines`）、人物关系（`character_cards`）、高能场面（`signature_scenes`）
- 往后：让分镜表能顺着剧本自然长出来，每个动作节点对应一个或多个镜头

剧本的核心作用是两件事：
1. 把大纲里的故事落成可执行的「场」—— 谁和谁在哪里发生了什么
2. 给分镜表提供稳定、可拆解的叙事单位

## 二、适用文件

1. `02_<剧名>_英文剧本.docx`
2. `06_<剧名>_中文剧本.docx`

## 三、Quick Reference 必须包含

1. 项目名
2. Logline
3. 目标市场
4. 类型
5. 集数
6. 单集时长
7. 核心情绪/视觉标签
8. 主角速览

---

## 四、主体结构

### 4.1 整体层级

```
EPISODE（集）
  ├── [EPISODE BRIEF]  ← 集头摘要，第一场之前
  └── SCENE（场次）× 2-4
        ├── 场次头信息（7 字段）
        ├── [OPENING STATE]
        ├── [MAIN BODY]（动作块序列）
        └── [SCENE ENDING]
```

### 4.2 单集场次数量与时长分配

| 单集时长 | 建议场次数 | 每场时长目标 |
|---------|-----------|------------|
| 1 min | 2-3 场 | 20-35s / 场 |
| 1.5 min | 3-4 场 | 20-35s / 场 |
| 2 min | 3-5 场 | 25-40s / 场 |

**最低硬性要求：每集不少于 2 场。**

**与分镜表的密度咬合逻辑：**

```
剧本：1 场（[OPENING STATE] + 5-7 个动作块 + [SCENE ENDING]）
  ↓
分镜表：[OPENING STATE] → 1-2 建立镜
        每个动作块 → 1-3 shot rows
        [SCENE ENDING] → 1-2 收场镜
  ↓
合计：1 场 ≈ 6-12 shot rows ≈ 20-45s 画面时长

全集 2-4 场 × 20-45s = 40-180s → 覆盖 1-2 min 单集
```

每场动作块写得足，分镜表自然能填满时长。不需要人为增加场次数量。

## 五、[EPISODE BRIEF]（集头摘要）

**位置**：紧接在 `## EPISODE XX` 之后、第一个 `### SCENE` 之前。

**6 个字段，全部必填，各 1-3 句：**

| 字段（英文剧本标签） | 说明 |
|------------------|------|
| `Episode Plot` | 本集剧情摘要（2-4 句，覆盖全集所有场次） |
| `Opening Hook` | 开场靠什么留住观众（冲突/信息/画面，一句话） |
| `Cold Open Focus` | 前 3-5 秒画面上必须有什么（具体可视元素，一句话；供分镜前几镜对齐） |
| `Closing Hook` | 结尾留给下一集的悬念或危机（一句话；须与末场 `[SCENE ENDING]` 一致） |
| `Core Satisfaction` | 本集核心爽点类型 + 落点（一句话；对齐大纲 `satisfaction_beat`） |
| `Target Emotion` | 观众到片尾应处于什么情绪（一句话） |

**约束：**
- `Opening Hook` / `Cold Open Focus` 必须在**第一场**前半段兑现
- `Closing Hook` 必须与**末场** `[SCENE ENDING]` 同一 beat
- `Episode Plot` 必须覆盖本集**全部场次**，不得只写第一场

---

## 六、场次头信息（每场必填，7 字段）

```
### SCENE XX.XX
Time: [Day / Night / Dawn / Dusk / Pre-Dawn / Continuous]
Location: [INT./EXT. 地点名]
Characters: [本场全部出场角色]
Core Event: [本场最核心的叙事事件，一句话]
Scene Result: [本场结束时局面如何变化，一句话]
Outline Ref: [本场主要兑现大纲哪个字段，如 core_event / turn / emotional_goal / relationship_shift]
```

**`Outline Ref` 说明：**

| 值 | 对应大纲字段 | 含义 |
|----|------------|------|
| `core_event` | `episode_outlines.core_event` | 本场是本集的主线事件场 |
| `turn` | `episode_outlines.turn` | 本场承载本集的核心转折 |
| `emotional_goal` | `episode_outlines.emotional_goal` | 本场负责推动或兑现情绪目标 |
| `relationship_shift` | `episode_outlines.relationship_shift` | 本场记录关系变化 |
| `antagonist_pressure` | `episode_outlines.antagonist_pressure` | 本场是反派/机构的压力落点 |
| `satisfaction_beat` | `episode_outlines.satisfaction_beat` | 本场是爽点兑现场 |
| `setup` | — | 本场纯粹为后续场次铺垫 |

每集所有场次的 `Outline Ref` 合起来，必须覆盖大纲本集 `core_event` 和 `turn` 两个字段（不允许这两项没有对应场次）。

**作用：** 场次头信息 7 个字段直接输出到分镜表 `scene_index`，两份文件天然一一对应。

---

## 七、[OPENING STATE]（开场状态）

1-4 句，不写散文，必须覆盖：

| 内容 | 说明 |
|------|------|
| 环境状态 | 空间、光线、氛围 |
| 人物初始状态 | 人在哪里、做什么、什么姿态 |
| 关键道具 | 本场会发挥作用的道具，**首次出现必须在这里写清楚位置** |
| 当下局面 | 这场戏开始时的权力/情绪状态 |

**作用：** 分镜表前 1-2 个 shot（建立镜、道具插入镜）从这里直接拆。

---

## 八、[MAIN BODY]（动作块正文）

### 8.1 动作块分段规则

按「动作块」分段，每块只处理一个连续动作或一次短交锋。

**触发换段的情况（每种情况独立成段）：**

| 情况 | 说明 |
|------|------|
| 人物进入或离开 | 单独一行 |
| 任何可视化动作（拿、推、站、起、转身） | 独立段落 |
| 对白前后的动作/反应 | 对白前写动作，对白后写反应 |
| 局面变化（权力转移/信息揭露/情绪转折） | 独立段落 |
| 关键道具出现或被使用 | 独立句，明确位置 |

**每个动作块 ≈ 1-3 个分镜 shot rows。**

### 8.2 最低动作块数（防缩水，与分镜密度咬合）

| 场次类型 | 最低动作块数 | 对应分镜 shot 预估 |
|---------|------------|-----------------|
| 对话主导场 | 5 块 | 6-10 shots |
| 冲突/对抗场 | 7 块 | 9-14 shots |
| 情绪爆发/反转/揭露场 | 6 块 | 8-12 shots |
| 快速过场（≤20s） | 3 块 | 4-6 shots |

这些数字与分镜表规范的「每场最低 shot 数」咬合：动作块 × 平均 1.5 shot/块 ≈ 分镜 shot 数。

### 8.3 对白格式

```
CHARACTER NAME
Dialogue line.

CHARACTER NAME (O.S.)
Dialogue from off screen — character is in the scene but not in frame.

CHARACTER NAME (V.O.)
Narration — character is not physically present in this scene.

CHARACTER NAME (CONT'D)
Same character continues after an action line interruption.

CHARACTER NAME
(parenthetical — tone or direction, use minimally)
Dialogue line.
```

**标注使用规则：**

| 标注 | 使用条件 | 禁止用法 |
|------|---------|---------|
| `(O.S.)` | 角色在场景里但镜头外说话 | 电话另一端（用 `(V.O.)` 或直接描述） |
| `(V.O.)` | 旁白、内心独白、录音播放 | 正常对话 |
| `(CONT'D)` | 同一角色被动作行打断后继续对白 | 无打断时 |
| Parenthetical | 仅当语气从文字无法判断时使用 | 解释情绪（情绪写进动作行）|

**Parenthetical 使用上限：全场不超过 3 个括弧说明。** 情绪和语气优先通过动作行表达，括弧只做最后兜底。

### 8.4 COLD OPEN（冷开场，选用）

如果本集开头有独立的前情提要或冷开场序列（pre-title sequence），在 `[EPISODE BRIEF]` 之后、第一个 `### SCENE` 之前，用以下格式标注：

```
### COLD OPEN
Time: ...
Location: ...
Characters: ...
Core Event: ...
Scene Result: ...
Outline Ref: setup

[OPENING STATE]
...

[MAIN BODY]
...

[SCENE ENDING / END COLD OPEN]
```

冷开场不计入正常场次编号（不写 `SCENE 01.XX`），但必须符合四段结构要求。时长目标：15-25s。

---

## 九、[SCENE ENDING]（场尾钩子）

单独标注，**不能埋在正文里**，**不能用平淡叙述结束**。

必须是以下四种类型之一：

| 类型 | 说明 |
|------|------|
| `cliffhanger` | 危机动作，不给解决 |
| `reversal` | 反转对白或揭露行，落地即停 |
| `emotional_peak` | 高强度情绪动作行，留白 |
| `setup` | 预告下一场/下一集危险的动作或对白 |

**每集末场的 `[SCENE ENDING]` 必须是集末钩子**，且与 `[EPISODE BRIEF]` 的 `Closing Hook` 一致。

## 十、对白质量规则（内嵌版，完整版见 `rules/08_英文对白写作规则.md`）

### 10.1 每句对白必须做到其中一件事

| 功能 | 说明 |
|------|------|
| **推进冲突** | 改变对抗态势，让局面更紧或出现意外 |
| **揭露信息** | 给出观众或另一角色之前不知道的事 |
| **改变情绪** | 转换场内情绪方向（从对抗到脆弱；从委屈到愤怒） |

**不满足以上任何一项的对白行，删掉或合并进其他行。**

### 10.2 禁止重复已知信息

上一句已经说清的内容，下一句不允许重复表达（不管换了角色还是换了说法）。

### 10.3 角色声音必须可区分

同一场里不同角色的对白风格不能完全一样。每个主要角色需要有稳定的语言特征（句长/句式/用词倾向），在场次内保持一致。

### 10.4 短句优先

高压冲突场、情绪爆发场：单句对白不超过 15 个词。解释性信息场：拆成多轮交互，不要一口气说完。

### 10.5 禁止项（对白层）

| 禁止 | 原因 |
|------|------|
| 中式英语直译腔 | 见 `rules/08_英文对白写作规则.md` 常见错误示例 |
| 长篇解释型独白（>3 句连续单角色） | 分镜无法拆，节奏死 |
| 书面作文式对白 | 不像人说的话 |
| 全场所有角色说话风格一样 | 人物无法区分 |
| 对白解释刚发生过的动作 | 动作行已经写了，不要重复 |

## 十一、可视化写作原则（动作行层）

### 11.1 少写抽象感受，多写可被拍出来的东西

| 禁止写 | 改写方向 |
|--------|---------|
| 她很痛苦 | 她攥着手机的手发白 |
| 他很震惊 | 他看完照片后，半天没说话 |
| 空气仿佛凝固 | 房间里没人动，只有空调声 |
| 她感到命运不公 | 她嘴上还在笑，眼泪已经掉下来 |

**原则：分镜只能拍能看见的东西。每一句动作行，都必须能在画面上找到对应。**

### 11.2 关键道具必须有明确落点

凡是本场会被拆成镜头的道具，必须在 `[OPENING STATE]` 或首次出现时写清楚位置。

### 11.3 转折点必须写实

以下转折必须在动作行里明确写出，不能靠对白暗示：谁先沉默 / 谁先变脸 / 谁先起身 / 谁把东西推过去或拿回来 / 谁打断谁 / 谁突然进门。

### 11.4 每个动作块的三问自检

```
□ 现在画面里发生了什么？（可见）
□ 谁在主动推动局面？（清晰）
□ 这块结束后，局面有什么变化？（有推进）
```

## 十三、禁止项（完整版）

**剧本内容层：**
1. 禁止省略 `[EPISODE BRIEF]` 或其中任一字段
2. 禁止 `[EPISODE BRIEF]` 与正文场次「两张皮」（钩子/悬念必须在正文兑现）
3. 禁止每集没有场次覆盖大纲的 `core_event` 和 `turn`
4. 禁止概述式正文（「二人争吵后和解」类写法）
5. 禁止一场包含多个空间或多个情绪波峰
6. 禁止 `[SCENE ENDING]` 缺失或以平淡叙述替代
7. 禁止用「以此类推」或摘要替代任何集的完整正文

**动作行层：**
8. 禁止把剧本写成小说（大量心理描写/抒情/抽象比喻）
9. 禁止裸台词（对白前后没有动作或反应行）
10. 禁止关键道具位置不明确
11. 禁止转折点（沉默/变脸/起身）只靠对白暗示

**对白层：**
12. 禁止写 shot size、机位、运镜、转场、AI 备注（属于分镜表）
13. 禁止中式英语直译腔
14. 禁止长篇解释型独白（连续 3 句以上同一角色无打断）
15. 禁止全场角色对白风格无区分
16. 禁止对白重复刚发生过的动作行内容

## 十四、输出前自审

```
□ 全剧每一集均有 [EPISODE BRIEF]，6 个字段均已填写
□ [EPISODE BRIEF] 与第一场兑现开场/冷开场焦点，与末场兑现结尾悬念
□ 全剧每一集均有完整正文，无摘要替代
□ 每集场次数量在单集时长对应的建议范围内（1min→2-3场；2min→3-5场）
□ 每场有完整 4 段结构（头信息 7 字段 + OPENING STATE + MAIN BODY + SCENE ENDING）
□ 每场 Outline Ref 已填写；全集场次合起来覆盖大纲 core_event 和 turn
□ 每场 [MAIN BODY] 动作块数量达到场次类型最低标准（对话场≥5 / 冲突场≥7）
□ 每场 [OPENING STATE] 包含环境/人物状态/关键道具/当下局面
□ 关键道具在首次出现时有明确位置
□ 每场 [SCENE ENDING] 单独标注，明确使用四种类型之一
□ 每集末场 [SCENE ENDING] 与 [EPISODE BRIEF] Closing Hook 一致
□ 全场无裸台词，无概述式段落
□ 对白均满足：推进冲突 / 揭露信息 / 改变情绪 三者之一
□ 无长篇解释型独白，角色语言风格可区分
□ Parenthetical 全场不超过 3 个
□ scene_crosswalk 与正文场次编号完全一致
□ scene_crosswalk 每条含 time / core_event / scene_result / outline_ref
```
