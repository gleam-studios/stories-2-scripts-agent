# 12 剧本规范

## 一、适用文件

1. `02_<剧名>_英文剧本.docx`
2. `06_<剧名>_中文剧本.docx`

## 二、Quick Reference 必须包含

1. 项目名
2. Logline
3. 目标市场
4. 类型
5. 集数
6. 单集时长
7. 核心情绪/视觉标签
8. 主角速览

## 三、主体格式

固定格式：

`EPISODE -> SCENE -> INT./EXT. -> 动作行 -> 角色名 -> 对白`

### 场景格式

```text
## EPISODE 01

### SCENE 01.01
INT. LOCATION - NIGHT

Action line in present tense.

CHARACTER NAME
Dialogue line.
```

## 四、强制要求

### 4.1 格式要求

1. 每场都必须有 `scene_id`。
2. 所有 slugline 必须用 `INT./EXT.`。
3. 角色名必须大写。
4. 剧本正文不得缺少动作行和对白。

### 4.2 动作行密度要求（防缩水核心规则）

每一场的动作行必须覆盖以下三个功能层，缺一不可：

| 功能层 | 说明 |
|-------|------|
| **空间建立** | 一句确立场景、人物位置和整体氛围的动作行 |
| **关键物理动作** | 驱动场次叙事的核心动作（入场、拿取、碰撞、离开等） |
| **情绪次文本** | 一句揭示角色内心状态、肢体语言或微表情的动作行 |

单集时长为 1-2 min 时，每场动作行总量建议不少于 **3 行**（不含 slugline）。

### 4.3 对白密度要求

1. 每场至少 **2 轮有效对白交换**（来回各算一次）。
2. 高压冲突场、情绪爆发场、身份揭露场不得少于 **3 轮**。
3. 每集不得有超过 1 场"纯动作无对白场"（用于特殊节奏节点，如追逐、逃跑）。

### 4.4 集末钩子要求

每集最后一场的结尾，必须满足以下之一：

| 钩子类型 | 说明 |
|---------|------|
| `cliffhanger` | 以危机行动行或悬念动作收尾，不给解决 |
| `reversal` | 以一句反转对白或揭露行收尾 |
| `emotional_peak` | 以一个高强度情绪动作行收尾（不解释，留余韵）|
| `setup` | 以一个明确预告下集危险的对白或动作行收尾 |

**禁止以平淡对白或普通描述性动作行结束一集。**

### 4.5 全剧覆盖要求

1. 必须为全剧每一集写完整剧本，不得用"以此类推"或"后续集数格式相同"等替代。
2. 每集必须有不少于 2 场戏。
3. `scene_crosswalk` 中的 `scene_id` 必须与正文 `SCENE` 编号完全对应，无遗漏。

## 五、禁止项

1. 禁止写 shot size、机位、运镜、转场、AI 备注。
2. 禁止把剧本写成小说叙述。
3. 禁止把设定集当作剧本前言整段复制。
4. 禁止某集只写场次索引或剧情摘要替代正文。
5. 禁止动作行只写地点描述而不含人物行为。
6. 禁止每集只有 1 场戏（除非明确标注为短序幕/收尾场）。

## 六、输出前自审

在提交剧本前，必须执行以下核查：

```
□ 全剧每一集均有完整正文（非摘要）
□ 每集至少 2 场，每场有 slugline + 动作行 + 对白
□ 每场动作行覆盖空间建立 / 物理动作 / 情绪次文本三个功能层
□ 每集末场以 cliffhanger / reversal / emotional_peak / setup 之一收尾
□ scene_crosswalk 与正文场次编号完全一致
□ 无中文直译腔对白
```

以上任意一项不通过，必须补写后再提交。
