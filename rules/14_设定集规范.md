# 14 设定集规范

## 一、适用文件

1. `04_<剧名>_英文设定集.docx`（由 `skills/english-setting-bible-builder` 生成）
2. `08_<剧名>_中文设定集.docx`（由 `skills/chinese-mirror-pack-translator` 镜像生成）

设定集是**全项目最重型的设计文档**，其功能是：为整个制作流程提供唯一的视觉标准、人物标准、资产生成标准。  
所有其他文件（大纲、剧本、分镜表）必须与设定集保持一致，而不是设定集去迁就它们。

---

## 二、结构总览（12 章，固定顺序）

| 章节 | 标题 | 最低数量要求 |
|------|------|--------------|
| §1 | Quick Reference（项目速览） | 必须完整 |
| §2 | World Background & Positioning（世界观与项目定位） | 5 个字段 |
| §3 | Timeline & Backstory（时间线与前史） | ≥ 3 条事件 |
| §4 | Character Profiles（角色详细设定） | ≥ 4 个角色，每角色 12 个字段 |
| §5 | Character Relationship Matrix（角色关系矩阵） | ≥ 3 对关系 |
| §6 | Look Profiles（造型系统） | 每主要角色 ≥ 1 套造型，每套 9 个字段 |
| §7 | Location Profiles（场景设定） | ≥ 3 个场景，每场景 9 个字段 |
| §8 | Prop Profiles（道具设定） | ≥ 3 个道具，每道具 7 个字段 |
| §9 | Visual System（整体视觉系统） | 5 个字段 |
| §10 | Continuity Rules（连续性规则） | ≥ 6 条，结构化 |
| §11 | AI Asset Matrix（AI 资产矩阵） | 每个 LOOK/LOC/PROP 各 1 条 |
| §12 | Narrative Forbidden Zones（叙事禁区） | ≥ 3 条 |

---

## §1 Quick Reference — 必填字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `project_title` | string | 剧集英文名 |
| `logline` | string | 1-2 句，含主角/核心冲突/赌注 |
| `target_market` | string | 目标市场（如 NA-English） |
| `genre` | array\[string\] | 主类型 + 子类型，如 \["Hidden Identity", "Revenge Romance"\] |
| `episode_count` | integer | 总集数 |
| `episode_duration` | string | 单集时长，如 "1-2 min" |
| `mood_tags` | array\[string\] | 3-5 个核心视觉/情绪标签 |
| `visual_identity_line` | string | 一句话描述整体视觉气质 |
| `protagonist_snapshot` | string | 1 句话描述主角的核心行动模式 |

---

## §2 World Background & Positioning — 必填字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `world_type` | enum | `contemporary` / `period` / `fantasy` / `sci-fi` |
| `power_structure` | string | 谁控制资源、地位、信息，规则是什么 |
| `social_rules` | string | 这个世界运行的隐性法则（什么行为被奖励/惩罚） |
| `the_core_wound` | string | 故事世界中那个根本的不公正或失衡 |
| `tone_summary` | string | 一句话定调，如 "cold luxury revenge with public humiliation as weapon" |

---

## §3 Timeline & Backstory — 事件格式

每条事件必须包含 3 个字段：

| 字段 | 说明 |
|------|------|
| `year_or_phase` | 时间点（如 "Five years ago" / "Present day" / "Episode 1" ） |
| `event` | 事件描述（具体、可视化） |
| `dramatic_impact` | 该事件对故事的戏剧性影响（推动什么冲突或信息） |

最低 3 条，推荐 5-8 条。

---

## §4 Character Profiles — 每个角色 12 个必填字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `character_id` | string | `CHAR-01` 格式，必须唯一 |
| `name` | string | 全名 + 别名（如有） |
| `role_type` | enum | `protagonist` / `antagonist` / `ally` / `rival` / `love_interest` / `neutral` |
| `public_identity` | string | 世界所见的身份 |
| `hidden_identity` | string | 仅观众/部分角色知道的真实身份（无则填 `none`） |
| `core_desire` | string | 最深层的驱动（主动追求什么） |
| `core_fear` | string | 最深层的恐惧（极力避免什么） |
| `fatal_flaw` | string | 可能毁掉自己的性格缺陷 |
| `character_arc` | string | `[start_state] → [end_state]` 格式 |
| `voice_style` | string | 3 个形容词，如 "short, cold, precise" |
| `signature_behaviors` | array\[string\] | 2-3 个可观察的具体习惯性行为（可被拍到的） |
| `visual_design` | string | 完整外形描述（身形、面部、气质） |
| `look_ids` | array\[string\] | 关联的 LOOK-XX |
| `signature_props` | array\[string\] | 关联的 PROP-XX |

最低 4 个角色（主角、主反派、盟友或爱人、次要推进角色各 1）。

---

## §5 Character Relationship Matrix — 每对关系 5 个字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `char_a` | string | CHAR-XX |
| `char_b` | string | CHAR-XX |
| `relationship_type` | string | 如 "rival sisters" / "hidden allies" / "forced marriage" |
| `hidden_tension` | string | 双方之间未说出口的核心张力 |
| `power_balance` | enum | `a_dominant` / `b_dominant` / `equal` / `shifting` |

最低 3 对（至少含主角-主反派、主角-爱人）。

---

## §6 Look Profiles — 每套造型 9 个必填字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `look_id` | string | `LOOK-01` 格式 |
| `character_id` | string | CHAR-XX |
| `look_name` | string | 简短名称，如 "Power Black" / "Disguise Casual" |
| `look_function` | enum | `power` / `vulnerable` / `disguise` / `casual` / `formal` / `confrontation` |
| `episodes_or_contexts` | string | 何时使用（如 "Episodes 1-10, public gala scenes"） |
| `color_palette` | array\[string\] | 3-5 个颜色名称或 hex |
| `key_garments` | array\[string\] | 服装构成列表 |
| `hair_and_makeup` | string | 发型 + 妆容描述 |
| `ai_prompt_seed` | string | 直接可用于 AI 图生成的 prompt 种子 |

---

## §7 Location Profiles — 每个场景 9 个必填字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `location_id` | string | `LOC-01` 格式 |
| `name` | string | 场景名 |
| `type` | enum | `interior` / `exterior` / `vehicle` / `virtual` |
| `dramatic_function` | string | 这个空间在故事中承担什么戏剧功能 |
| `visual_keywords` | array\[string\] | 4-6 个视觉关键词 |
| `lighting_mood` | string | 光线/氛围描述 |
| `sound_environment` | string | 声音环境描述（用于 sfx_music 参考） |
| `continuity_notes` | string | 需要在所有场次保持一致的细节 |
| `ai_prompt_seed` | string | 直接可用于 AI 图生成的 prompt 种子 |

最低 3 个场景。

---

## §8 Prop Profiles — 每个道具 7 个必填字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `prop_id` | string | `PROP-01` 格式 |
| `name` | string | 道具名称 |
| `owner_char_id` | string | CHAR-XX 或 `shared` |
| `story_function` | enum | `evidence` / `symbol` / `weapon` / `personal_item` / `plot_device` / `status_marker` |
| `first_appears_episode` | integer | 首次出现的集数 |
| `visual_description` | string | 详细外观（材质/颜色/尺寸/状态） |
| `continuity_rules` | array\[string\] | 该道具需保持一致的具体细节 |
| `ai_prompt_seed` | string | 直接可用于 AI 图生成的 prompt 种子 |

最低 3 个道具。

---

## §9 Visual System — 5 个必填字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `overall_color_palette` | object | `dominant` / `accent` / `shadow` / `highlight` 四个色槽 |
| `lighting_philosophy` | string | 全片光线哲学，如 "cold top light, minimal fill" |
| `texture_language` | array\[string\] | 3-5 个质感标签 |
| `hero_vs_villain_contrast` | object | `hero_visual` / `villain_visual` 各一句话对比说明 |
| `forbidden_aesthetics` | array\[string\] | 绝对禁止的视觉风格（如 "warm filter", "pastel softness"） |

---

## §10 Continuity Rules — 结构化格式

每条规则必须包含 4 个字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `rule_id` | string | `CONT-01` 格式 |
| `rule_type` | enum | `character` / `prop` / `location` / `costume` / `timeline` |
| `applies_to` | string | 对应 CHAR-XX / PROP-XX / LOC-XX / LOOK-XX |
| `rule_description` | string | 具体规则描述 |
| `violation_severity` | enum | `blocking`（阻断性错误） / `warning`（警告） |

最低 6 条，覆盖至少 2 个角色、1 个道具、1 个场景。

---

## §11 AI Asset Matrix — 每条 6 个字段

每个 LOOK-XX / LOC-XX / PROP-XX 都需要一条对应 AI 资产记录：

| 字段 | 类型 | 说明 |
|------|------|------|
| `asset_id` | string | 对应 LOOK/LOC/PROP ID |
| `asset_type` | enum | `character_look` / `location` / `prop` / `vfx` |
| `prompt_seed` | string | 完整可用的 AI 生成 prompt |
| `must_have` | array\[string\] | 必须包含的视觉元素列表 |
| `must_avoid` | array\[string\] | 必须排除的视觉元素列表 |
| `resolution_hint` | enum | `portrait` / `landscape` / `square` |

---

## §12 Narrative Forbidden Zones — 叙事禁区

字符串数组，每条描述一个在任何剧本/分镜中都绝对不能出现的叙事或视觉行为。  
最低 3 条，推荐 5-8 条。  
例如：
- "Nora must never beg, cry first, or physically retreat from a confrontation."
- "The family seal must never appear damaged or incomplete."

---

## 五、最低数量红线

| 对象 | 最低数量 |
|------|---------|
| 角色（有完整 12 字段） | 4 |
| 造型（有完整 9 字段） | 4（每主角≥1） |
| 场景（有完整 9 字段） | 3 |
| 道具（有完整 7 字段） | 3 |
| 连续性规则 | 6 |
| 关系矩阵条目 | 3 |
| AI 资产矩阵条目 | ≥ LOOK+LOC+PROP 总数 |
| 叙事禁区 | 3 |

---

## 六、禁止项

1. 禁止把设定集写成剧本摘要（没有戏剧事件叙述，只有设定信息）
2. 禁止只写 2-3 条简略人物介绍就交付
3. 禁止 `character_profiles` 中任何角色缺少 `character_arc` 或 `signature_behaviors`
4. 禁止 `look_profiles` 中缺少 `ai_prompt_seed`
5. 禁止 `ai_asset_matrix` 中的 `must_have` 和 `must_avoid` 使用单一字符串（必须是数组）
6. 禁止 `continuity_rules` 只写无结构的散文列表（必须用 rule_id + rule_type 结构）
7. 禁止 `visual_system` 中 `forbidden_aesthetics` 缺失（空数组不合格）
8. 禁止把"要求真实图片"作为交付前提

- [ ] `narrative_forbidden_zones` ≥ 3 条
