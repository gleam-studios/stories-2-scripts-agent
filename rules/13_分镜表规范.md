# 13 分镜表规范

## 一、定位

分镜表是整个生产链的**最下游执行文档**。

- 往前：从剧本（`scene_crosswalk` + 场次正文）精确映射，内容和剧情走向不得脱离剧本和大纲
- 往后：直接输入镜头生成、图像生成、视频生成、表演控制、配音/字幕/剪辑节奏

分镜表不是「镜头清单」，而是**以「集」为总节奏单位、以「场」为事件单位、以「分镜」为连续画面任务单位，对剧本进行逐层可视化拆解的执行文档**。

## 二、适用文件

1. `03_<剧名>_英文分镜表.docx`
2. `07_<剧名>_中文分镜表.docx`

## 三、三级结构（固定，不得跨级）

```
EPISODE（集）— 总节奏 + 单集钩子
  └── SCENE（场）— 一个完整戏剧事件
        └── SHOT（分镜）— 连续画面任务最小单位
```

### 3.1 第一级：集级总信息（每集开头必写）

每集 shot table 之前，先输出集级摘要块：

```
## EPISODE XX

Episode Core Event:   （本集核心叙事事件，一句话）
Episode Tone:         （本集情绪基调：如「压抑 / 不安 / 关系撕裂」）
Key Reversal:         （本集关键反转，一句话）
Closing Hook:         （本集结尾钩子，一句话；与剧本 [EPISODE BRIEF].Closing Hook 对齐）
```

### 3.2 第二级：场级信息（每场 shot table 之前必写）

```
### SCENE XX.XX — [场次简称]
Time:             （Day / Night / Dawn / Dusk / Pre-Dawn / Continuous）
Location:         （INT./EXT. 场景名）
Characters:       （本场全部出场角色）
Core Event:       （本场核心事件，一句话）
Start State:      （本场起点：人物状态 + 当前权力/情绪格局）
End State:        （本场终点：人物状态变化 + 格局如何改变）
Scene Type:       （场次类型枚举，见 §5.2）
Screenplay Ref:   （对应剧本 scene_crosswalk 的 scene_id，如 S01.03）
```

`Start State` 和 `End State` 共同定义这场戏的「入场」和「出场」，是拆镜的边界。

### 3.3 第三级：分镜（shot rows）

每条分镜 = 一个明确的**连续画面任务单元**，只负责一件主要的事。

## 四、分镜必填字段（21 列）


| 字段                       | 说明                                                         |
| ------------------------ | ---------------------------------------------------------- |
| `shot_id`                | 格式 `集.场.S镜序`，如 `01.03.S01`；可从集→场→镜三级回溯                     |
| `episode_id`             | 如 `EP01`；与集级摘要块一致，供自动化流水线索引                                |
| `scene_id`               | 如 `S01.03`；必须与 `scene_index` 和剧本 `scene_crosswalk` 一致      |
| `shot_type`              | 分镜功能类型（枚举，见 §5.1）                                          |
| `screenplay_ref`         | 对应剧本正文哪一句动作行或对白（可写动作行原文摘录或「SCENE 01.03 OPENING STATE 第1句」） |
| `visual_content`         | 这一镜**画面里具体看见什么**；只写可视动作，不写抽象理解或剧情总结                        |
| `character_action`       | 本镜中角色的主要动作（谁做了什么；用于表演/运动控制）                                |
| `key_info_point`         | 这一镜要让观众知道什么；决定这条镜头是否必要                                     |
| `shot_language`          | 景别 / 机位 / 运镜 / 构图重点，合并成一条字符串（如 `WIDE                       |
| `dialogue_anchor`        | 本镜对应的对白原句，或 `None`                                         |
| `emotion`                | 本镜主情绪（一镜一主情绪，如 `Maya: suppressed / Elliot: alert`）         |
| `rhythm`                 | 节奏标注（枚举，见 §5.3）                                            |
| `estimated_duration_sec` | 预计时长，如 `3s` / `2-4s`；单集累计须落在 60-120s                       |
| `transition`             | 与下一镜的衔接方式（枚举，见 §5.4），不得自由书写                                |
| `sfx_music`              | 本镜的声音策略：环境声 / 音乐进入或退出 / 静音 / 重音点                           |
| `continuity_notes`       | 连续性说明：角色位置、道具状态、服装状态、屏幕亮灭、血迹/湿痕等，供自动化生产追踪                  |
| `character_ids`          | 本镜角色 ID 列表，与设定集对齐                                          |
| `location_id`            | 场景 ID，与设定集对齐                                               |
| `prop_ids`               | 本镜出现的道具 ID 列表                                              |
| `look_ids`               | 本镜角色造型 ID 列表                                               |
| `ai_note`                | AI 生图/生视频专用备注；不得留空（见 §7）                                   |

## 五、枚举定义

### 5.1 `shot_type`（分镜功能类型）


| 值                       | 含义                               |
| ----------------------- | -------------------------------- |
| `environment_establish` | 环境交代——建立空间、氛围、道具位置               |
| `character_enter`       | 人物出场——引入角色，交代初始状态和位置             |
| `action_advance`        | 动作推进——角色执行关键物理动作（拿/推/起身/进门等）     |
| `dialogue_delivery`     | 对白承接——说出对白；必须有前置动作镜              |
| `emotion_reaction`      | 情绪反应——接收信息后的情绪/态度变化（只有价值反应才单独成镜） |
| `info_reveal`           | 信息揭示——道具/画面/文本呈现关键叙事信息（如手机屏幕内容）  |
| `reversal`              | 转折——权力关系/信息状态/情绪方向在这一镜发生明确翻转     |
| `hook`                  | 钩子——场尾/集尾的悬念落点，必须单独成镜，不得埋入正文     |


每场 shot table 中 `**hook` 类型的分镜必须存在**（场尾钩子）；`**environment_establish` 必须是该场第一个或前两个分镜**。

### 5.2 `scene_type`（场次功能类型，用于场级信息块）


| 值                                | 最低 shot 数 | 预期 shot 数 |
| -------------------------------- | --------- | --------- |
| `dialogue_driven`                | 4         | 6-10      |
| `confrontation_conflict`         | 6         | 9-14      |
| `emotional_peak_reversal_reveal` | 5         | 8-12      |
| `quick_transition`               | 2         | 4-6       |


**全季保底**：`shot_rows 总数 ≥ 全剧场次总数 × 4`

**与剧本密度咬合**（见 `rules/12_剧本规范.md §4.2a`）：

```
目标 shot 数 = screenplay_pack.scene_crosswalk[scene_id].action_block_count × 1.5（向上取整）+ 2
实际 shot 数 ≥ max(目标 shot 数, 该 scene_type 最低 shot 数)
```

### 5.3 `rhythm`（节奏标注）


| 值                    | 含义                 | 典型时长 |
| -------------------- | ------------------ | ---- |
| `fast_cut`           | 快切——快速信息传递，冲突密集段   | 1-2s |
| `normal`             | 正常节奏——标准对话覆盖       | 2-4s |
| `pause`              | 停顿——沉默/反应/让观众消化    | 3-5s |
| `sustained_pressure` | 拉长压迫——紧张感蓄积，镜头持续停留 | 4-8s |


### 5.4 `transition`（转场枚举）


| 值              | 含义              |
| -------------- | --------------- |
| `CUT`          | 普通硬切            |
| `HARD_CUT`     | 强硬切入，力度高于普通 CUT |
| `SMASH_CUT`    | 情绪或信息突然撞切，常用于钩子 |
| `MATCH_CUT`    | 动作或构图匹配切        |
| `REACTION_CUT` | 切到反应镜           |
| `HOLD`         | 本镜停留，不立即切出      |
| `DISSOLVE`     | 叠化，用于时间或情绪过渡    |
| `FADE_OUT`     | 淡出，通常用于集尾或段落结束  |

## 六、场次骨架（每场必须覆盖的功能锚点）

每场戏必须包含以下 5 种功能锚点分镜，不得缺失：


| 锚点       | 对应 `shot_type`                              | 位置        |
| -------- | ------------------------------------------- | --------- |
| **开场建立** | `environment_establish` + `character_enter` | 该场前 1-2 镜 |
| **触发动作** | `action_advance`                            | 冲突开始时     |
| **核心对抗** | `dialogue_delivery` + `emotion_reaction` 交替 | 场中段       |
| **转折点**  | `reversal`                                  | 场中后段      |
| **场尾钩子** | `hook`                                      | 该场最后 1 镜  |


快速过场（`quick_transition`）可压缩至：`environment_establish` + `hook`，但仍必须填写完整 21 列，不得省略声音或连续性字段。

`**quick_transition` 豁免说明：** 触发动作、核心对抗、转折点三个锚点对 `quick_transition` 场次**不强制要求**。这类场次的唯一硬性要求是首镜 `environment_establish` + 末镜 `hook`，总 shot 数 ≥ 2。不得以「这是过场」为由省略 `hook` 镜。

## 七、7 条拆镜规则

### 规则 1：一条分镜只承担一个主要功能

不要在一条分镜里同时承担：环境交代 + 揭示关键道具 + 完成情绪反转 + 说完三句重要对白。镜头会糊。

### 规则 2：画面焦点变化 → 考虑拆新镜

以下情况通常需要拆：关注对象改变 / 动作主体改变 / 信息重点改变 / 情绪主导者改变 / 空间位置改变 / 说话权转移。

### 规则 3：戏剧节点变化 → 拆镜

以下节点是天然分镜点：**进入 / 停顿 / 看见 / 触碰 / 递出 / 起身 / 靠近 / 打断 / 反问 / 愣住 / 闯入 / 摔落 / 揭示**。

每一个节点单独一条分镜，不要与前后动作糊在一起。

### 规则 4：对白按「冲突单位」切，不按句机械切

不是每句话就一个镜头。判断标准：

- 这句是否改变了局面？→ 拆
- 这句是否值得看反应？→ 拆（并加 `emotion_reaction` 跟镜）
- 这句是否是关键信息点？→ 拆
- 这句是否是压迫或钩子？→ 拆

普通应答可与动作放同一镜；反杀式台词 / 信息炸弹台词必须单独拆，并给对方反应镜。

### 规则 5：反应镜头只在有价值时存在

以下情况值得单独 `emotion_reaction` 分镜：情绪明显变化 / 认知发生变化 / 真相被击中 / 权力关系逆转 / 钩子落在反应上。

不值得的反应（普通点头/应答）合并进对白镜，不单独成镜。

### 规则 6：场尾钩子必须单独成镜、单独标注

`hook` 类型的分镜不得埋在 `dialogue_delivery` 或 `action_advance` 里。它必须是独立的最后一行，`transition` 写 `HARD_CUT` 或 `SMASH_CUT`，`rhythm` 写 `pause` 或 `sustained_pressure`。

### 规则 7：服务 AI 可生成性

AI 流程额外约束：

- 不要在一镜里塞多个复杂连续动作（一个主动作/镜）
- 不要让角色状态在同一镜内跳变太大（先/后帧要可连续）
- 同一场内角色服装、位置、道具状态须可追踪（写 `prop_ids` / `look_ids`）
- `continuity_notes` 必须明确记录影响后续镜头的一切状态变化（如“手机屏亮屏并停留在偷拍视频界面”“右手拿卡，左手拿照片”）
- `ai_note` 必须写成可直接输入生成系统的描述：包括光线方向、色调、构图参考、角色外观锚点

## 八、shot sequence 参考模板

每场 shot 展开的参考骨架（可压缩合并，不可跳过建立镜和情绪镜）：

```
1. WIDE / ESTABLISHING  — 空间与人物关系建立（environment_establish）
2. MEDIUM / CHARACTER   — 人物初始状态（character_enter 或 action_advance）
3. MEDIUM TWO-SHOT      — 双人对话或对峙覆盖（dialogue_delivery）
4. MCU / CLOSE          — 主角反应/情绪推进（emotion_reaction）
5. INSERT / DETAIL      — 道具、眼神、手部等关键细节（info_reveal，如适用）
6. REVERSAL SHOT        — 权力/信息翻转（reversal）
7. WIDE / HOOK FRAME    — 场尾钩子构图（hook）
```

## 九、时长预算


| 分镜类型        | 建议时长     |
| ----------- | -------- |
| 建立宽镜        | 3-6s     |
| 对话中景覆盖      | 2-4s / 句 |
| 情绪近景        | 2-5s     |
| 信息插入（道具/屏幕） | 1-3s     |
| 过场/转场       | 1-2s     |
| 钩子镜（停顿压迫）   | 3-6s     |


单集所有 shot 的 `estimated_duration_sec` 累计应落在 **60-120s** 区间。

---

## 十、禁止项

1. 禁止每场只写 1 个 shot（除明确标注的 `quick_transition`）
2. 禁止把整场剧情写成散文说明替代 shot table
3. 禁止缺失资产 ID（`prop_ids` / `look_ids` / `location_id`）
4. 禁止 `visual_content` 写剧情摘要——只写画面里能看见的东西
5. 禁止 `ai_note` 留空——每个 shot 必须给出可服务 AI 生图或生视频的具体备注
6. 禁止 `hook` 类型分镜缺失——每场必须有场尾钩子
7. 禁止 `environment_establish` 不是该场的第一或第二镜
8. 禁止分镜内容脱离剧本——`screenplay_ref` 字段必须能回指到对应场次的具体动作块或对白行
9. 禁止 `emotion_reaction` 滥用——普通应答不单独成镜
10. 禁止集级总信息块缺失或只写场次 shot，忽略集级摘要
11. 禁止 `transition` 自由书写，必须使用 §5.4 枚举值
12. 禁止 `sfx_music` 留空——至少说明环境声底、音乐变化或刻意静音
13. 禁止 `continuity_notes` 留空——必须记录关键状态连续性

## 十一、输出前自审

```
□ 每集开头有集级总信息块（Episode Core Event / Tone / Key Reversal / Closing Hook）
□ 每场开头有场级信息块（8 字段含 Start State / End State / Scene Type / Screenplay Ref）
□ shot_rows 总数 ≥ 场次总数 × 4
□ 每场 shot 数量 ≥ 该场 scene_type 最低标准
□ 每场 shot 数量 ≥ action_block_count × 1.5 + 2（取自 screenplay_pack.scene_crosswalk）
□ 每场包含 5 种功能锚点（开场建立 / 触发动作 / 核心对抗 / 转折点 / 场尾钩子）
□ 每场第一或第二镜是 environment_establish
□ 每场最后一镜是 hook，且单独成行
□ 全部 21 列字段均已填写，无空缺
□ screenplay_ref 每条均可回指到剧本具体句子或动作块
□ transition 全部使用 §5.4 枚举值，无自由文本
□ sfx_music 每条均写明环境声/音乐/静音策略
□ continuity_notes 每条均写明关键状态连续性
□ ai_note 每条均有实质性生成描述，含光线/色调/构图/角色外观锚点
□ estimated_duration_sec 单集累计在 60-120s 内
□ 全剧所有 scene_id 在 scene_index 中均有对应
□ 全剧所有 scene_id 在 shot_rows 中均有覆盖，无遗漏场次
□ [SCENE ENDING] 类型与该场最后 hook 镜的 transition 对齐（cliffhanger → `SMASH_CUT` / `HARD_CUT`）
```
