# 16 长文本分段执行规范

## 一、适用场景

当英文原文体量达到或接近 **~50 万字符 / ~12 万 token** 以上，或单次调用无法在保证输出余量的前提下装入「原文 + 规则 + 指令」时，**必须**按本章执行分段协议。  
对外仍只报告 **7 个 stage**；分段是 **stage 内部子步骤**，不改变交付物清单。

**禁止：** 将全书原文一次性塞入单次模型调用（除非已实测当前模型上下文与输出余量足够，且通过自检）。

---

## 二、工作目录约定（`work/<project_slug>/`）

| 路径 | 说明 |
|------|------|
| `source_raw.txt`（或项目约定的单一原文文件） | 完整原文唯一真源 |
| `chapter_index.yaml`（推荐） | 每章：`chapter_id`、标题、`start_line`/`end_line` 或 `char_start`/`char_end`，或 `anchor_start`/`anchor_end`（起止锚点子串） |
| `source_chunks/`（可选） | 预切文本块，如 `ch_01-05.txt`，便于重复调用与审计 |
| `story_bible_partials/`（可选） | 每 chunk 的 `story_bible_partial.json` |
| `run_manifest.json` | 见 §八 |

---

## 三、Chunk 默认参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 按章包大小 | **3–8 章 / 包** | 章很短可调高，章很长可调低 |
| 按 token 估算 | **每包约 15k–25k tokens（输入侧）** | 为规则与输出预留 ≥30% 窗口 |
| 重叠 | **0–1 章** | 仅当拆包处可能切断情节时使用 1 章重叠 |

单包输入 = 该区间原文 + 本 stage 必需规则摘要（非全书规则）。

---

## 四、`source-analysis` 子步骤

### 4.1 Chunk pass

每包一次模型调用，输出 **`story_bible_partial`**（JSON），至少包含：

- `chunk_id`：如 `CH01-CH05`
- `chapter_range`：章节 ID 列表或起止编号
- `characters_seen`：本包出现的角色要点（可后并入 `character_cards`）
- `plot_beats`：情节节点短列表
- `open_threads`：未解决线头，供 merge 对齐
- 以及与本包相关的 `worldbuilding_seeds` / `timeline_seeds` / `location_candidates` / `prop_candidates` 等**子集**（字段名可与完整 `story_bible` 对齐，便于合并）

输入需附带 `rules/10_相似度评分规则.md` 的**执行摘要**（由总控摘录要点即可，不必全文）。

### 4.2 Merge pass

**单独一次调用**，输入 = **全部** `story_bible_partial` JSON 数组（**不含原文**），输出 = 符合 `story_bible.json` schema 的 **`story_bible.json`**。

**必须**写入 `source_coverage`（见 `skills/source-analysis/schema.json`）：

- `chapters_covered`：已覆盖章节列表（与 `chapter_index` 一致）
- `chunk_ids_merged`：参与合并的 `chunk_id` 列表
- `merge_version`：如 `1.0`

**Merge 规则：**

1. **ID 冲突**：同一实体以 **首次出现 chunk 的 `character_id` / `location_id` / `prop_id` 为准**；后续 chunk 仅合并为别名说明或合并到同一 ID 的备注字段，**禁止** silent 改 ID。
2. **禁止编造**：merge 不得引入 **任一 chunk 的 `plot_beats` / `supporting_evidence` 均未支撑** 的主要情节或主要角色。
3. **缺口**：若某章区间无 partial，必须 **补跑该 chunk** 或标明 `coverage_gaps` 并补全后再进入 `overseas-adaptation-planner`。

## 五、`english-outline-writer` 子步骤

### 5.1 Batch pass

- **第一批**：大纲 §1–§8（至 `Overall Story Structure`），不含分集细表或仅含前若干集占位（按 SKILL 约定）。
- **后续批**：仅 **`episode_outlines`**（及依赖集纲的附录若 SKILL 要求），每批建议 **10 集**（可调），直到 `adaptation_plan.episode_count` 满。

每批输入必须包含 **Continuity Sheet**（见 §七），**不得**附带完整 `setting_bible.json` 全文；仅摘录硬约束。

### 5.2 Align pass

单独调用：输入 = 合并后的 `outline_pack` 草案 + `adaptation_plan`（+ 可选 Continuity Sheet）。

**只做：** 矛盾消除、钩子/卡点与集纲对齐、缺集补全、编号连续。

**禁止：** 无依据重写已冻结的 Quick Reference / Core Selling Points（除非与 `adaptation_plan` 显式冲突并在输出中注明修正原因）。

**校验：** `episode_id` 从 1 到 `episode_count` 连续、无重复。

---

## 六、`english-screenplay-writer` 子步骤

- **默认每集 1 次**模型调用；最长不超过 **每调用 3 集**（若单集极短且总控自检通过可调宽）。
- 每轮输入：`outline_pack` 中该集条目、`adaptation_plan` 摘要、**Continuity Sheet**、本集涉及 LOC/PROP/LOOK 列表、**原文仅附** `chapter_index` 对应摘录或 `source_chunks` 中的相关块。
- 增量写入 `screenplay_pack.json`；最后 **Stitch pass**（可脚本或单次模型）：`scene_id` 全局唯一、`scene_crosswalk` 连续、跨集衔接无断档。

---

## 七、`english-storyboard-table-builder` 子步骤

- **按集**生成 `shot_rows`；单集场次多或镜头公式结果过大时 **按场**拆分调用。
- 每轮输入：该集/场 `screenplay` 片段 + 对应集 `outline` 摘要 + `setting_bible` 中 ID 对照表（非全文）。
- **密度 second pass**：对照 `rules/13` §4.2a `target_shot_count = ceil(action_block_count × 1.5) + 2`；不足则 **仅补镜头行**，不重写剧本。

**禁止：** 在常规集数下 **单次调用生成全剧分镜表**；除非集数极少且通过 `rules/13` 自检。

---

## 八、`run_manifest.json` 建议字段

```json
{
  "project_slug": "string",
  "completed_steps": ["source_analysis_merge", "outline_batch_2"],
  "chunk_ids_done": ["CH01-CH05", "CH06-CH10"],
  "outline_batches": ["ep01-10", "ep11-20"],
  "last_episode_screenplay": "EP12",
  "last_storyboard_scene": "S12.03",
  "notes": "string"
}
```

旧工作目录若已使用 `outline_batches_done`，与 `outline_batches` **同义**；新稿统一写 `outline_batches`。

**新对话续跑：** 总控先读 `run_manifest.json` 与 `soul.md`，再从断点继续。

---

## 九、`chinese-mirror-pack-translator` 与 `english-setting-bible-builder`

- **镜像**：与英文相同的 **集/批边界** 分批翻译，合并为最终 `mirror_pack`；禁止单次塞入四份英文终稿全文若已超上下文。
- **设定集**：`look_profiles` / `location_profiles` / `prop_profiles` / `ai_asset_matrix` 可 **分批生成 + 一次 merge JSON**，merge 仅去重与 ID 对齐，禁止改 `rules/14` 必填结构。

## 十一、自检清单（长文跑通）

- [ ] `chapter_index` 与 `source_coverage.chapters_covered` 一致
- [ ] `story_bible` 已 merge，无未覆盖章节
- [ ] `outline_pack` 经 align，`episode` 数量等于 `adaptation_plan.episode_count`
- [ ] `screenplay` 已 stitch，`scene_id` 无重复
- [ ] 分镜按集/场通过密度公式或 second pass
- [ ] `run_manifest.json` 与当前进度一致
