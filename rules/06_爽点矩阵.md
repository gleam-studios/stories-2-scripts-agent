爽点矩阵规则

来源：
1. `short-drama原始参考/satisfaction-matrix.md`

改写目的：
把原有短剧爽点设计规则改造成适合海外短剧改编的爽点规划工具。

一、核心原则

爽点的本质是：
压抑 -> 释放

没有前期压抑的爽点，不够值钱；
没有后续升级的爽点，也不够持久。

二、优先使用的 5 类爽点

1. 身份碾压
适用：
   - Hidden Identity
   - Billionaire
   - Secret Heir

2. 打脸复仇
适用：
   - Revenge
   - Betrayal
   - Power reversal

3. 逆袭翻盘
适用：
   - Second Chance
   - Underdog rise

4. 情感爆发
适用：
   - Romance
   - Flash Marriage
   - Enemies to Lovers

5. 悬念揭秘
适用：
   - Mystery
   - Secret Baby
   - Hidden Truth

三、爽点分布原则

1. 全剧至少覆盖 3 类以上爽点。
2. 同类型爽点不要连续过多使用。
3. 大爽点前必须有压抑蓄力。
4. 最强爽点放在最后阶段。

四、阶段性爽点建议

1. 起势段
特点：
   - 小爽点
   - 主要用于建立观众期待

2. 攀升段
特点：
   - 中型爽点增加
   - 让观众形成持续满足

3. 风暴段
特点：
   - 高频大爽点
   - 爽点与反转结合

4. 决战段
特点：
   - 极爽点
   - 终极兑现

五、爽点设计禁忌

1. 爽点被立刻推翻
2. 同一种爽点连续三次以上
3. 爽点强度不递增
4. 全剧最强爽点不在结尾阶段
5. 没有压抑就直接释放

六、英文剧本中的爽点落法

英文爽点不要写成解释式总结。

应通过以下方式体现：
1. 角色地位反转
2. 信息压制被打破
3. 情绪压抑突然释放
4. 观众早就期待的结果终于兑现

七、爽点自检清单

1. 爽点前是否存在蓄力？
2. 爽点释放是否彻底？
3. 爽点是否和人物核心欲望相关？
4. 爽点后是否还有后续推进空间？

八、接入建议

2. `english-outline-writer`
3. `english-screenplay-writer`
