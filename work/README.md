# work 说明

这个目录用于在 Codex 对话里手工跑单篇网文时，保存本次工作的中间产物和草稿。

## 推荐结构

每篇作品一个子目录：

`work/<project_slug>/`

例如：

`work/midnight-heiress/`

## 推荐文件

1. `source_packet.json`
2. `story_bible.json`
3. `adaptation_plan.json`
4. `setting_bible.json`
5. `outline_pack.json`
6. `screenplay_pack.json`
7. `storyboard_pack.json`
8. `mirror_pack.json`
9. `01_英文大纲.md`
10. `02_英文剧本.md`
11. `03_英文分镜表.md`
12. `04_英文设定集.md`
13. `05_中文大纲.md`
14. `06_中文剧本.md`
15. `07_中文分镜表.md`
16. `08_中文设定集.md`
17. `export/`
   - 存放本项目最终 8 个 Word 文件的项目内副本

## 原则

1. 这里可以保存工作过程文件。
2. 这里不是最终正式交付目录。
3. 最终正式交付只保留 8 个 `.docx` 文件。
4. 导出时还要额外把 8 个 Word 文件复制到：
   - `~/Desktop/短剧导出/<project_slug>/`
